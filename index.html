Privacy Policy — Hermes Personal Digest

Last updated: 2026-05-20

This application (“the App”) is a single-user, personal automation tool operated by its owner for private research and personal information digests. This policy explains what the App does with data.

Who this is for

The App is used solely by its owner. It is not a public product and has no other end users.

What data the App accesses

Using the Meta Threads API, the App reads public content returned by the Threads keyword search endpoint, limited to the following fields: id, text, permalink, timestamp, and username.

The App does not collect private messages, contact lists, email addresses, passwords, payment information, or any sensitive personal data.

How the data is used

  • Public posts matching a small fixed set of topic keywords are retrieved on a schedule.
  • Results are deduplicated and summarized into a short text digest.
  • The digest is delivered only to the owner's own private messaging channel (Telegram).

Data retention and sharing

  • Retrieved posts are processed transiently to generate the digest and are not retained beyond the run, except for an optional private archive in the owner's personal notes, accessible only to the owner.
  • The App does not sell, rent, or share any data with third parties.
  • The App does not use the data for advertising or profiling.

Data deletion

Because no third-party user data is stored, there is no shared dataset to delete. The owner can remove the local archive and revoke the App's access token at any time via the Meta App Dashboard.

Security

Access tokens and credentials are stored in environment configuration on a private server reachable only over the owner's private network (Tailscale).

Compliance

The App respects the Threads API rate limits and the Meta Platform Terms.
